# Resume Writing Guide

Reference material for the Job Application Tailor. Not auto-loaded into prompts. Use this to copy sections into Pipeline Editor system prompt fields for custom pipelines, or as a reference when tuning prompt templates.

---

## Core Principles for Human-Sounding Resumes

1. **Show, don't tell** — Instead of "passionate about security," describe a problem you solved.
2. **Use concrete details** — Real tools, real incidents, real outcomes.
3. **Avoid empty buzzwords** — Words like "synergy," "results-driven," "think outside the box" add nothing.
4. **Let achievements speak** — Numbers and before/after comparisons are powerful.
5. **Write in your natural voice** — Imagine explaining your work to a colleague.
6. **Be honest about your role** — Don't claim credit for team efforts unless you led them.
7. **Focus on impact, not just duties** — What changed because of your work?
8. **Keep sentences varied in length** — Too many uniform sentences sound robotic.

## General Resume Tips

### Experience Descriptions

- Start bullets with strong action verbs (avoid "responsible for" or "tasked with").
- Quantify whenever possible: "reduced incident response time by 40%" instead of "improved incident response."
- Include context: "During a company-wide migration to AWS, I..."
- Mention scale: "Supported 5,000+ endpoints" or "managed firewall rules for 50+ offices."
- Describe challenges: "Legacy systems made patching difficult, so I..."
- Use the STAR method (Situation, Task, Action, Result) naturally.
- Avoid clichés like "team player" — show collaboration through examples.
- Mention tools by name: "used Wireshark to analyze packet captures" not "used network analysis tools."
- Include both successes and lessons learned (humanizes you).
- Use present tense for current job, past for previous roles.
- Keep bullets to one or two lines; recruiters scan.
- Start with the most impressive achievement for each role.
- Don't list every task — curate the most relevant.
- Include projects you initiated, not just assigned tasks.
- Mention cross-team collaboration: "worked with developers to integrate security into CI/CD."
- Describe how you handled ambiguity or incomplete information.
- Use "we" for team efforts, but clarify your specific contribution.
- Avoid exaggerating: "led" vs "participated in" — be accurate.
- Include metrics like dollars saved, time saved, incidents prevented.
- Mention training or mentoring others — shows leadership.
- Reference specific frameworks or standards you applied (NIST, ISO 27001).
- Include any recognition or awards received.
- Describe troubleshooting processes you improved.
- Mention documentation you created or updated.
- Talk about automation you implemented to reduce manual work.
- Include any public speaking, presentations, or blog posts.
- Mention contributions to open source or internal tools.
- Describe how you stayed current with emerging threats.
- Include any security research or bug bounty findings.
- Mention process improvements that reduced friction for others.

### Skills Section

- Group skills into categories (e.g., "Security Tools," "Programming Languages," "Frameworks").
- Be specific: "Python (Flask, Pandas)" not just "Python."
- Avoid rating skills with stars or bars — just list them.
- Order by relevance to the job you're applying for.
- Include soft skills, but back them up in experience bullets.
- Don't list outdated tools unless you still use them.
- Mention familiarity with cloud platforms: AWS, Azure, GCP.
- Include both offensive and defensive security skills if applicable.
- List compliance frameworks you've worked with (GDPR, HIPAA, PCI-DSS).
- Mention any languages beyond English if relevant.
- Avoid generic "Microsoft Office" — assume it's a given.
- Include skills you're actively developing (shows growth mindset).
- Be honest about proficiency — "familiar with" vs "expert in."
- List relevant certifications (but put them in a separate section).
- Mention methodologies (Agile, Scrum, Kanban) if used.
- Include any SIEM, EDR, or SOAR platforms you've used.
- List scripting languages: Bash, PowerShell, Python.
- Mention cloud security tools: AWS Security Hub, Azure Defender.
- Include any GRC tools if applicable.
- Note if you have experience with specific regulations (SOX, FedRAMP).

### Education and Certifications

- List degrees with institution, year, and field of study.
- Include relevant coursework if you're a recent grad.
- List certifications with full name and issuing body (e.g., CISSP, OSCP).
- Include expected completion date for in-progress certs.
- Mention any academic honors or scholarships.
- Include continuing education or training courses.
- List relevant workshops or conferences attended.
- Mention any teaching or tutoring experience.
- Don't list high school if you have a college degree.
- Keep it concise — no need to list every course.
- Include any published research or papers.
- Mention if you hold any security clearances (if applicable).
- List any professional memberships (ISSA, OWASP, etc.).

### Formatting and Length

- Use a clean, ATS-friendly format (avoid columns, graphics).
- Keep font consistent and professional (Calibri, Arial, etc.).
- Use standard section headings: Summary, Experience, Skills, Education.
- Make the resume LONG — at least 2 full pages. Never shorten. Add bullets from base resume/context if needed to fill 2 pages.
- Bullet counts (minimums): first job (current) 10 bullets; second job 6; third job 5. Include every relevant bullet; prefer more rather than fewer.
- Under each job: no blank lines between bullet points; one blank line only before the next company.
- SUMMARY AT TOP (required): Right after your name, you MUST have a paragraph of 2–4 sentences before **Technical Expertise:**. Do not go straight from name to Technical Expertise. This summary must tie the job's requirements (required_skills, responsibilities — what they're looking for) to your real experience so the reader sees the match immediately.
- Use bullet points, not paragraphs, for experience.
- Ensure dates are clearly formatted (Month YYYY – Month YYYY).
- Use bold for company names and job titles, not all caps.
- Maintain consistent spacing and margins.
- Save as PDF unless instructed otherwise.
- Avoid photos, icons, or colors that might confuse ATS.
- Use reverse chronological order.
- Include a short intro (2–4 sentences) after your name: base it on the job posting's requirements/qualifications (what they're looking for) and tie each to your real experience — e.g. "They want X; I did X at [Company] when…" so the reader sees how you match the role immediately.
- Tailor resume to each job — highlight relevant keywords.
- Proofread for typos and grammatical errors — read aloud.

### Authenticity and Voice

- Write like you speak — avoid overly formal language.
- Use first person implied (no "I" in bullets, but it's understood).
- Avoid jargon you wouldn't use in conversation.
- Include small, human details: "When the SIEM went down at 3 AM, I..."
- Show personality through word choice, not generic phrases.
- Don't try to sound like a thesaurus — simpler is better.
- Vary sentence structure — mix short and long.
- Include a brief mention of a hobby or interest (optional, humanizes you).
- Avoid overusing "managed" or "led" — use a variety of verbs.
- Be honest about gaps — explain briefly (e.g., "parental leave," "travel").
- Don't inflate job titles — use your actual title.
- If you're early in career, focus on potential and eagerness to learn.
- If you're experienced, focus on wisdom and mentorship.
- Use "we" for team achievements, but clarify your part.
- Avoid buzzwords like "thought leadership" unless you actually have it.
- Mention specific challenges you overcame.
- Let your unique career path show — it's okay to not be linear.

### Cybersecurity Skills — Authentic Representation

- List tools you've actually used hands-on, not just heard of.
- Mention versions if relevant: "Splunk Enterprise 8.x."
- Distinguish between "used" and "expert in" — be honest.
- Include both commercial and open-source tools.
- Mention programming languages relevant to security: Python, Go, C, Assembly, PowerShell, Bash.
- List cloud platforms with specific services: AWS IAM, GuardDuty, Azure Defender.
- Include frameworks: MITRE ATT&CK, NIST CSF, ISO 27001.
- Mention methodologies: threat modeling (STRIDE, DREAD), risk assessment (FAIR).
- List specific attack types you've handled: phishing, ransomware, DDoS, insider threat.
- Include defensive technologies: WAF, IDS/IPS, HIDS/NIDS.
- Mention forensics tools: FTK, EnCase, Autopsy, Volatility.
- List malware analysis tools: IDA Pro, Ghidra, x64dbg, Cuckoo.
- Include network protocols: TCP/IP, HTTP/S, DNS, BGP.
- List databases: SQL, NoSQL, and their security aspects.
- Include container security: Docker, Kubernetes, Aqua, Twistlock.
- Mention DevSecOps practices: SAST, DAST, IAST, RASP.
- List compliance knowledge: GDPR, HIPAA, PCI-DSS, SOX, FedRAMP.
- Mention industry standards: OWASP Top 10, CWE/SANS Top 25.
- List specific cloud security tools: Prisma Cloud, Wiz, Orca.
- Include IAM tools: Okta, Azure AD, CyberArk, BeyondTrust.
- Mention threat intelligence platforms: MISP, ThreatConnect, Anomali.
- List vulnerability management tools: Qualys, Tenable, Rapid7.
- Include network security tools: Snort, Suricata, Zeek, tcpdump.
- Mention SIEM platforms: Splunk, QRadar, ArcSight, ELK.
- Include EDR/EPP: CrowdStrike, SentinelOne, Microsoft Defender.
- List scripting languages for log analysis: awk, sed, grep, jq.
- Mention hardware security: HSMs, TPM, secure boot.

### Quantifying Security Impact

- Reduce mean time to detect (MTTD): "from 24h to 1h."
- Decrease false positives: "by 50%" — shows efficiency.
- Prevent breach costs: "avoided $500k in potential breach costs."
- Increase patch compliance: "from 70% to 95%."
- Respond to incidents: "50 incidents with 0 repeat occurrences."
- Save person-hours: "200 person-hours per month through automation."
- Block volume: "10,000+ malicious emails monthly."
- Reduce attack surface: "by decommissioning 100 unused accounts."
- Achieve audit success: "100% audit pass rate for three consecutive years."
- Improve security awareness: "phishing click-through rate from 15% to 3%."
- Detect threats: "5 zero-day threats using threat hunting."
- Reduce patch time: "from 30 days to 7 days for critical vulnerabilities."
- Increase coverage: "SIEM coverage to 100% of critical assets."
- Maintain uptime: "99.9% uptime for security tools during incidents."
- Train at scale: "500+ employees, reducing phishing click rate from 15% to 3%."

## How to Use This Guide in the App

### Integrate into Custom Pipelines

In the Pipeline Editor (`/pipeline`), you can copy sections of this guide into the **System Prompt** field of any step:

- For a "Draft" step: paste the STYLE RULES and CONTENT RULES sections into the system prompt.
- For a "Review" step: paste the "Phrases to Avoid" table and tell the model to flag every occurrence.
- For a "Polish" step: paste the "Human Voice Check" principles and the banned phrase replacements.

### Quantification Prompts

When the model is reviewing a draft, include this in the system prompt for the review step:

```
For every achievement bullet, ask: How many? How much time? What percentage? What dollar value?
If the answer is unknown, flag it with "(add metric here)".
```

### Cybersecurity Tool Check

Add to the review step system prompt:
```
Flag any bullet that uses a generic category name instead of a specific tool.
Examples of flags: "vulnerability scanner" → ask for tool name, "SIEM" → ask for platform name, "EDR" → ask for vendor name.
```

### ATS-Friendly Output

Add to the final polish step system prompt:
```
Ensure the output uses clean plain text structure. No tables, no columns, no icons.
Use standard section headings: Summary, Skills, Experience, Education.
```
